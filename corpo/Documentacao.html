Documentação do Sistema

Bem-vindo ao Sistema de Gestão Casa Tapera

Esta documentação fornece uma visão geral da arquitetura, funcionalidades e casos de uso do sistema de gestão da Casa Tapera. O sistema foi projetado para ser uma ferramenta completa, abrangendo desde o gerenciamento de mesas e comandas em tempo real até a análise de dados financeiros e de vendas.

Casos de Uso Principais

Os casos de uso descrevem as interações entre os usuários (atores) e o sistema para atingir um objetivo específico. Abaixo estão detalhados os fluxos mais importantes da aplicação.

Caso de Uso: Gerenciar Comanda de Mesa

  • Ator Primário: Garçom / Operador de Caixa
  • Resumo: Este caso de uso abrange o ciclo de vida completo de uma comanda, desde a adição de itens até o fechamento da conta.
  • Cenário Principal (Fluxo de Sucesso) 1. O Garçom seleciona uma mesa Ocupada na tela de distribuição.
    2. O sistema exibe os detalhes da comanda (itens e total).
    3. O Garçom clica em "Adicionar Item" e seleciona um produto no modal do cardápio.
    4. O sistema adiciona o item, recalcula o total e atualiza a visualização da comanda.
    5. Ao final, o Garçom clica em "Fechar Conta".
    6. O sistema abre o modal de pagamento.
    7. O Garçom seleciona o método (Dinheiro, Cartão ou Pix).
    8. O sistema registra a venda no caixa, limpa a comanda e altera o status da mesa para Livre.
  • Cenários Alternativos - Alterar Quantidade: O Garçom utiliza os botões "+" ou "-" ao lado de um item para ajustar sua quantidade, e o sistema recalcula o total automaticamente.
    - Remover Item: O Garçom clica no botão "×" para remover um item da comanda, confirma a ação, e o sistema o retira da lista, atualizando o total.
  • Cenário de Exceção - Se o sistema não conseguir salvar os dados no navegador (ex: `localStorage` cheio), uma mensagem de erro pode ser exibida, e as alterações não serão persistidas.

Caso de Uso: Gerar QR Code de Mesa

  • Ator Primário: Gerente
  • Resumo: Permite ao gerente visualizar e baixar QR Codes individuais para cada mesa, que direcionam os clientes para o cardápio digital.
  • Cenário Principal (Fluxo de Sucesso) 1. O Gerente clica no botão "Gerenciar QR Codes".
    2. O sistema exibe um modal com a lista de mesas cadastradas.
    3. O Gerente seleciona uma mesa da lista (ex: "Mesa 05").
    4. O sistema gera e exibe o QR Code para a URL da mesa selecionada na área de visualização.
    5. O Gerente clica em "Baixar QR Code".
    6. O sistema inicia o download de um arquivo de imagem PNG (ex: `QRCode-Mesa-05.png`).
  • Cenários Alternativos - Mesa recém-criada: Após criar uma nova mesa no modal "Gerenciar Mesas", ela aparecerá automaticamente na lista do gerenciador de QR Codes, pronta para ter seu código gerado.
  • Cenário de Exceção - Se a biblioteca externa (`qrcode.js`) não for carregada por falha de conexão, o sistema exibirá um alerta de erro ao tentar visualizar o QR Code, e a geração falhará.
